Desvende o conceito de fluxo de dados unidirecional no React e seus benefícios na organização e previsibilidade do código.?

Definição: No React, o conceito de fluxo de dados unidirecional significa que os dados fluem em uma única direção, do componente pai para os componentes filhos. Os componentes pais passam dados para os filhos através de propriedades (props), e esses dados são imutáveis para os componentes filhos. Se um componente filho precisar alterar os dados, ele deve solicitar a alteração ao componente pai, que então ajusta seu estado e repassa os dados atualizados.
